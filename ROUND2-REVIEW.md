# AI 小白分享页 · 第二轮设计验收与修改清单

## 总判断
首版已经具备可演示基础：
- 手机优先方向正确
- 赛博朋克气质基本成立
- 翻页逻辑已经成型
- 章节结构可用

但目前仍偏“高保真原型”，还没到“可放心转发给非 AI 圈同事”的完成度。第二轮目标不是推翻，而是做 **体验降噪 + 节奏重构 + 内容表达增强**。

---

## 一、必须修改（P0）

### 1. AI 核心概念卡组：从左右滑动改成上下阅读
**问题：**
- 横向滑动会和整页翻页心智冲突
- 手机用户对多层横滑容忍度低
- 当前概念学习路径不够自然，容易中途退出

**修改方向：**
- 去掉横向 snap card 逻辑
- 改成纵向概念流 / 分组堆叠卡片
- 一屏控制在 2–4 个概念
- 每个概念统一结构：
  - 概念名
  - 一句话人话解释
  - 一个工作场景例子
- 可以拆成 2 屏甚至 3 屏来承载，避免一屏过满

**建议分组：**
- 第 1 组：AI / LLM / 多模态 / Prompt
- 第 2 组：Token / Context / Fine-tuning / Inference
- 第 3 组：RAG / Hallucination / Workflow / Tools / Agent

---

### 2. 文案密度整体压缩一轮
**问题：**
- 有些屏信息量偏大，像在手机里读讲义
- 当前更适合“认真阅读”，不够适合“演示浏览 / 被动观看”

**修改方向：**
- 每屏只保留 1 个核心观点
- 单屏正文尽量控制在：
  - 1 个核心标题
  - 3–5 个短 bullet
  - 每 bullet 最多 1 行半到 2 行
- 长段落改成短句、短列点、关键词强调
- 让用户 3 秒内知道这一屏在讲什么

**文案压缩原则：**
- 删掉解释性重复句
- 少讲“背景铺垫”，多讲“结论 + 例子”
- 一屏只留一个记忆点

---

### 3. 章节节奏要拉开
**问题：**
- 目前部分章节在信息密度和情绪节奏上太接近
- 翻页时缺少“轻—重—轻—重”的呼吸感

**修改方向：**
- 增加章节之间的节奏差异：
  - 有的屏偏“定义”
  - 有的屏偏“类比”
  - 有的屏偏“案例”
  - 有的屏偏“提醒”
- 在重信息章节后，插入更轻的总结/对比型页面
- 把“概念解释页”和“判断/趋势页”做明显分层

**建议节奏：**
1. 封面（情绪）
2. 为什么懂 AI（动机）
3. 概念组 1（扫盲）
4. 概念组 2（扫盲）
5. 大模型为什么聪明（理解）
6. Agent 是什么（重点）
7. Agent 为什么火（趋势）
8. 应用场景（落地）
9. 企业踩坑（冷静）
10. 入门建议（行动）
11. 总结 CTA（传播）

---

## 二、重要优化（P1）

### 4. 像素画要和内容更深绑定，不只是装饰
**问题：**
- 当前像素画氛围到位，但多数更像“场景陪衬”
- 还没真正服务信息表达

**修改方向：**
每一屏的像素画都要承担一个表达任务，而不是纯点缀。

**建议：**
- “为什么懂 AI” → 像素白领 + AI 助手同屏，体现协作关系
- “大模型为什么聪明” → 像素大脑 + 数据流 / 语言碎片
- “Agent 是什么” → 像素机器人手里连接工具面板
- “Agent 应用场景” → 不同职业角色像素人
- “企业踩坑” → 警示图标 + broken workflow
- “总结页” → 从办公桌到未来城市的视觉隐喻

**目标：**
用户扫一眼图，就大概知道这一屏讲什么。

---

### 5. 封面页更强一点
**问题：**
- 当前封面已可用，但“分享页入口感”还不够强
- 少了一个能让人愿意继续翻下去的钩子

**修改方向：**
封面应更像一个可转发的“AI 入门特刊封面”。

**建议增强：**
- 增加更强副标题，例如：
  - 给完全不懂 AI 的同事，一次讲清核心概念与 Agent
- 增加 3 个醒目标签：
  - 名词扫盲 / Agent 原理 / 应用热点
- 增加“向上滑动开始”或更自然的进入引导
- 视觉中心更集中：机器人 / 终端 / 霓虹标题三者形成一个封面主视觉

---

### 6. 结尾 CTA 更像可转发页面
**问题：**
- 当前总结页更像结束语，不像传播页

**修改方向：**
结尾页要能承担“看完后愿意转发给同事”的作用。

**建议结构：**
- 一句强总结：
  - AI 不是一个新软件，而是一种新的工作方式
- 三条 takeaways：
  - 听懂核心概念
  - 理解 Agent 为什么火
  - 知道 AI 在工作中怎么落地
- 增加一句适合转发的话：
  - 如果你身边还有 AI 小白，这一页可以直接转给他

---

## 三、细节优化（P2）

### 7. 页面内视觉层级再强化
- 小标题和正文的区分可以更明显
- 关键词高亮策略要统一，避免一会儿黄一会儿青但没有规则
- 每屏控制主色重点，不要所有颜色一起抢戏

### 8. 减少“看起来像演示模板”的感觉
- 某些卡片排列还偏标准化
- 可以加入更明确的不对称布局或主次关系
- 某些屏可以采用“一大观点 + 两小补充”的版式，而不是平均分配

### 9. 适当补一个“人话类比”页
Agent 对小白仍然是最容易听懂但最容易误解的点。
可以考虑加入更明显的类比表达：
- Chatbot = 会回答问题的人
- Agent = 能接任务往前推进的人

这个类比适合做成独立屏，信息记忆度会更高。

## 四、给 COD 的明确执行要求

### 目标
在首版基础上做第二轮精修，不推翻技术方案，但重做部分内容结构与交互表达。

### 必改清单
1. 概念卡组取消左右滑动，改成纵向阅读结构
2. 文案整体压缩，单屏控制信息量
3. 拉开章节节奏，必要时重排页面顺序
4. 像素画与屏幕主题强绑定，不做纯装饰
5. 强化封面页
6. 强化结尾 CTA 页

### 交付要求
- 继续保持手机优先
- 继续保留赛博朋克方向，但克制一点，不要花里胡哨
- 不要增加复杂依赖
- 优先延续当前单文件或轻量静态结构
- 输出一版可直接预览的新版本
- 附一份“第二轮改了什么”的简短说明

---

## 五、验收标准
如果第二轮做好，应该满足下面几个感受：

- 用户不用学习交互，顺着滑就能看懂
- 一屏一个重点，不会觉得累
- 图不只是好看，而是帮理解内容
- 封面更像分享作品，结尾更像可传播页面
- 整体从“原型”升级到“可展示版本”
